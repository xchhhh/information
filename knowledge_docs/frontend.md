# 前端设计：聊天页与后台管理页

前端用原生 HTML / CSS / JavaScript 实现，不引入任何前端框架，零构建、由 FastAPI 同源托管。共两个完整页面：聊天页（index.html）与后台管理页（admin.html）。

## 一、聊天页（index.html）

访客打开 http://公网IP:8000 即可使用，无需登录。核心交互是卡片式对话：

- 用户消息与机器人回复分别以不同样式的"卡片"（bc-msg 类）呈现，左右分区、气泡化，比纯文本更直观；
- 每条机器人回复若带引用来源，会展示资料编号，便于追溯；
- 调用 POST /chat 获取答案，请求头带 X-API-Key 鉴权。

## 二、三态主题

页面支持浅色 / 深色 / 跟随系统三种主题：

- 浅色（light）、深色（dark）可由用户手动切换；
- 跟随系统（system）通过 CSS 的 prefers-color-scheme 自动匹配操作系统的明暗设置；
- 主题状态持久化（如 localStorage），刷新不丢失。

三态主题让页面在考官不同设备上都能舒适阅读，也体现"考虑了真实使用场景"的细节品味。

## 三、视觉风格：玻璃拟态

采用 Glassmorphism（玻璃拟态）：半透明面板 + 背景模糊 + 细边框 + 柔和阴影，配合极光渐变背景，整体不单调、有层次，比纯色块更现代。

## 四、磁吸按钮

关键操作按钮（如发送、上传）带"磁吸"微交互：鼠标靠近时按钮轻微向光标方向位移 / 放大，离开回弹。纯 CSS + 少量 JS 实现，提升点击欲望与精致感，且不影响功能。

## 五、拖拽上传

后台管理页支持把文件直接拖到上传区（drag & drop），松开即触发上传，比传统"点按钮选文件"更顺手。上传走 POST /admin/upload，服务端做类型白名单（.txt/.md/.pdf）与 20MB 大小限制、路径穿越防护。

## 六、为什么不用框架

个人作品集体量小、访问量低，原生实现的好处是：依赖为零、构建为零、加载最快、最易部署（FastAPI 直接返回静态文件）。把精力放在"交互细节"而非"工程脚手架"上，反而更显完成度。

## 七、同源部署

FastAPI 同时托管 API 与前端静态文件：根路径 / 返回聊天页，/admin 返回后台页，/ 下的前端资源由同一服务供给。无需额外 Nginx 或对象存储，一个进程搞定前后端，部署最简单。
